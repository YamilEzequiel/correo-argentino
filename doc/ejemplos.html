1 Instalación

Instalá la librería y axios (peer dependency).

npm install ylazzari-correoargentino axios
Requisitos Node.js ≥ 14. Credenciales de MiCorreo (userToken, passwordToken, email, password) — se piden por separado para TEST y PROD.

2 Inicialización

Hay dos formas de inicializar, según tengas o no el customerId.

Modo 1: con email + password (obtiene customerId automáticamente)

import CorreoArgentinoApi from "ylazzari-correoargentino";
import { Environment } from "ylazzari-correoargentino/enums";

const api = new CorreoArgentinoApi();

await api.initializeAll({
  userToken: process.env.USER_TOKEN!,
  passwordToken: process.env.PASSWORD_TOKEN!,
  email: process.env.EMAIL!,
  password: process.env.PASSWORD!,
  environment: Environment.TEST,
});

console.log("customerId:", api.getVarCustomerId());

Modo 2: con customerId ya conocido (salta /users/validate)

import CorreoArgentinoApi from "ylazzari-correoargentino";
import { Environment } from "ylazzari-correoargentino/enums";

const api = new CorreoArgentinoApi();

await api.initializeWithCustomerId({
  userToken: process.env.USER_TOKEN!,
  passwordToken: process.env.PASSWORD_TOKEN!,
  customerId: "0090000025",
  environment: Environment.PROD,
});
Las credenciales son distintas por ambiente Las que te dan para TEST no funcionan en PROD y viceversa. Asegurate de usar las correctas.

3 Cotizar un envío

Cotiza un envío a domicilio con un solo producto.

import { DeliveredType } from "ylazzari-correoargentino/enums";

const cotizacion = await api.getRates({
  customerId: api.getVarCustomerId(),
  postalCodeOrigin: "1757",
  postalCodeDestination: "1704",
  deliveredType: DeliveredType.D,
  dimensions: [
    { length: 30, width: 20, height: 10, weight: 1500, quantity: 1 },
  ],
});

console.log(cotizacion.rates);
// [
//   {
//     deliveredType: "D",
//     productType: "CP",
//     productName: "Paq.ar Clásico",
//     price: 498.06
//   }
// ]
Límites del server Peso 1–25000 g. Cada dimensión hasta 150 cm. Si pasás más, MiCorreo rechaza el request.

4 Cotizar varios productos en una sola request

La librería combina las dimensiones para vos. Pasá un array y arma el contenedor virtual automáticamente.

const cotizacion = await api.getRates({
  customerId: api.getVarCustomerId(),
  postalCodeOrigin: "1757",
  postalCodeDestination: "1704",
  deliveredType: DeliveredType.D,
  dimensions: [
    { length: 30, width: 20, height: 10, weight: 1500, quantity: 1 },
    { length: 20, width: 15, height: 5,  weight: 500,  quantity: 2 },
    { length: 10, width: 10, height: 5,  weight: 200,  quantity: 3 },
  ],
});

Cómo se calcula el contenedor virtual

CampoFórmula
weightsuma de weight × quantity de cada item
lengthmáximo de length × quantity
widthmáximo de width
heightsuma de height × quantity (items apilados)
¿Necesitás otro modelo de empaquetado? Si tu lógica de embalaje es distinta (ej: cubicación volumétrica real), calculá las dimensiones en tu código y pasá un único ProductDimensions con los valores finales.

5 Cotizar domicilio Y sucursal en una sola request

Omití deliveredType y MiCorreo te devuelve las dos modalidades en rates.

const cotizacion = await api.getRates({
  customerId: api.getVarCustomerId(),
  postalCodeOrigin: "1757",
  postalCodeDestination: "1704",
  // sin deliveredType
  dimensions: [{ length: 30, width: 20, height: 10, weight: 1500 }],
});

cotizacion.rates.forEach((r) => {
  console.log(`${r.deliveredType}: ${r.productName} → $${r.price}`);
});
// D: Paq.ar Clásico → $498.06
// S: Paq.ar Clásico → $398.06

6 Listar sucursales por provincia

Devuelve un array de agencias de la provincia indicada. Útil cuando vas a hacer un envío con deliveryType: "S" y necesitás el código de la sucursal destino.

import { ProvinceCode } from "ylazzari-correoargentino/enums";

const sucursales = await api.getAgencies(ProvinceCode["Provincia de Buenos Aires"]);

sucursales
  .filter((s) => s.status === "ACTIVE" && s.services.packageReception)
  .forEach((s) => {
    console.log(`${s.code} - ${s.name} (${s.location.address.postalCode})`);
  });
// B0107 - Monte Grande (B1842ZAB)
// B0312 - La Plata Centro (B1900)
// ...
Cambio en v0.1.0 Antes el tipo decía { agencies: Agency[] } pero el server siempre devolvió un array pelado. Ahora el tipo refleja la realidad: Agency[]. Si tu código usaba response.agencies, cambialo por response.

7 Crear un envío a domicilio

Una vez cotizado, creás el envío con shippingImport. extOrderId debe ser único — si lo repetís, MiCorreo rechaza.

import { DeliveredType } from "ylazzari-correoargentino/enums";

const envio = await api.shippingImport({
  customerId: api.getVarCustomerId(),
  extOrderId: "ORDER-12345",        // único — usá UUID o ID de tu DB
  orderNumber: "102",                // visible en el panel de MiCorreo

  recipient: {
    name: "María González",
    email: "maria@mail.com",
    phone: "1144556677",
    cellPhone: "1144556677",
  },

  shipping: {
    deliveryType: DeliveredType.D,
    address: {
      streetName: "Av. Corrientes",
      streetNumber: "1234",
      floor: "5",
      apartment: "B",
      city: "Buenos Aires",
      provinceCode: "B",
      postalCode: "1425",
    },
    weight: 1000,         // gramos (entero)
    declaredValue: 500.0,
    height: 20,           // cm (entero, ≤255)
    length: 40,
    width: 20,
  },
});

console.log("Envío creado:", envio.createdAt);
// "2026-05-14T16:15:04.996-03:00"
Idempotencia Guardá el extOrderId en tu DB antes de llamar a shippingImport. Si reintentás con el mismo ID, vas a recibir "La orden ya fue importada con anterioridad".

8 Crear un envío a sucursal

Para envíos a sucursal, en lugar de address pasás agency con el código (lo sacás de getAgencies).

const envio = await api.shippingImport({
  customerId: api.getVarCustomerId(),
  extOrderId: "ORDER-12346",

  recipient: {
    name: "María González",
    email: "maria@mail.com",
  },

  shipping: {
    deliveryType: DeliveredType.S,
    agency: "B0107",      // código de la sucursal, obtenido vía getAgencies
    weight: 1000,
    declaredValue: 500.0,
    height: 20,
    length: 40,
    width: 20,
  },
});
Validaciones client-side La librería valida antes de pegarle al server: si pasás deliveryType: "S" sin agency, o "D" sin address completa, te tira Error en local sin malgastar request.

10 Flujo completo de e-commerce

Caso real: tu tienda online recibe un pedido y necesita cotizar, mostrar opciones al cliente, y crear el envío cuando confirma.

import CorreoArgentinoApi from "ylazzari-correoargentino";
import { Environment, DeliveredType, ProvinceCode } from "ylazzari-correoargentino/enums";
import type { ShippingImport } from "ylazzari-correoargentino/interfaces";

// ───── 1. Inicialización (una vez al levantar el server) ─────
const api = new CorreoArgentinoApi();
await api.initializeAll({
  userToken: process.env.USER_TOKEN!,
  passwordToken: process.env.PASSWORD_TOKEN!,
  email: process.env.EMAIL!,
  password: process.env.PASSWORD!,
  environment: Environment.PROD,
});

// ───── 2. El cliente arma su carrito ─────
const carrito = [
  { sku: "ABC", length: 30, width: 20, height: 10, weight: 1500, quantity: 1 },
  { sku: "DEF", length: 20, width: 15, height: 5,  weight: 500,  quantity: 2 },
];

// ───── 3. Cotizar ambas opciones (D y S) para mostrar al cliente ─────
const cotizacion = await api.getRates({
  customerId: api.getVarCustomerId(),
  postalCodeOrigin: "1425",   // tu depósito
  postalCodeDestination: "1704", // dirección del cliente
  dimensions: carrito.map((p) => ({
    length: p.length,
    width: p.width,
    height: p.height,
    weight: p.weight,
    quantity: p.quantity,
  })),
});

// El cliente elige una opción y los datos de entrega.
const opcionElegida = cotizacion.rates.find((r) => r.deliveredType === "D");
if (!opcionElegida) throw new Error("Sin opciones de envío disponibles");

// ───── 4. Si elige sucursal, primero le mostramos las disponibles ─────
const sucursales = await api.getAgencies(
  ProvinceCode["Provincia de Buenos Aires"]
);
const sucursalesActivas = sucursales.filter(
  (s) => s.status === "ACTIVE" && s.services.packageReception
);

// ───── 5. Crear el envío ─────
const orderId = `ORDER-${Date.now()}`; // o tu ID interno

const envio: ShippingImport = {
  customerId: api.getVarCustomerId(),
  extOrderId: orderId,
  orderNumber: "TIENDA-102",
  recipient: {
    name: "María González",
    email: "maria@mail.com",
    cellPhone: "1144556677",
  },
  shipping: {
    deliveryType: DeliveredType.D,
    address: {
      streetName: "Av. Corrientes",
      streetNumber: "1234",
      city: "Buenos Aires",
      provinceCode: "B",
      postalCode: "1425",
    },
    weight: carrito.reduce((s, p) => s + p.weight * p.quantity, 0),
    declaredValue: 1500,
    height: 30,
    length: 40,
    width: 25,
  },
};

const resultado = await api.shippingImport(envio);

// ───── 6. Guardar en tu DB para evitar duplicados ─────
await db.orders.update(orderId, {
  shippingCreatedAt: resultado.createdAt,
  shippingProvider: "correo-argentino",
});
Persistí el extOrderId Es la clave de idempotencia. Si tu API rest hace retry de la request, querés evitar crear envíos duplicados.

11 Manejo de errores

Todos los métodos tiran Error con un mensaje del formato:

// 🔴 Error en {método}: {código} - {mensaje}
try {
  await api.shippingImport(envio);
} catch (err) {
  if (err instanceof Error) {
    console.error(err.message);
    // "🔴 Error en shippingImport: 402 - La orden ya fue importada con anterioridad."

    if (err.message.includes("orden ya fue importada")) {
      // Tratar como éxito idempotente: el envío ya existe en MiCorreo.
      return { alreadyCreated: true };
    }

    throw err;
  }
}

Errores típicos

HTTPCausa
400Falta un campo obligatorio
401JWT inválido (re-inicializá)
402Regla de negocio (ver mensaje específico)
404Recurso no existe (customerId equivocado, sucursal inexistente)
409Conflicto (idempotencia)
429Rate limit — implementá backoff exponencial
5xxError del lado de Correo

12 Activar modo debug

Por defecto la librería no imprime nada en consola. Si necesitás ver qué se está enviando, pasá debug: true al constructor.

const api = new CorreoArgentinoApi({ debug: true });

await api.initializeAll({ ... });
// 🌍 Ambiente configurado: https://apitest.correoargentino.com.ar/micorreo/v1
// 🔑 Token obtenido correctamente
// 🆔 CustomerId obtenido correctamente

await api.getRates({ ... });
// 🔍 Payload getRates: { customerId: "...", postalCodeOrigin: "...", ... }
NUNCA actives debug en producción Aunque la librería ya no logea el JWT (lo arreglamos en v0.1.0), los payloads de envíos contienen PII: emails, nombres, direcciones. Mantenelo desactivado en producción para no leakear datos personales en tus logs centralizados.
